# RoverMower

Автономная роботизированная косилка на базе мобильной платформы, RTK/GNSS-навигации и Raspberry Pi.

## О проекте

**RoverMower** — это проект по разработке автономной колесной косилки для работы на открытых участках.  
Цель проекта — собрать собственную платформу, которая сможет двигаться по заданному маршруту, ориентироваться с высокой точностью по RTK/GNSS и выполнять покос без постоянного участия оператора.

В основе идеи — объединить:

- механическую платформу повышенной проходимости;
- косильный модуль;
- высокоточную спутниковую навигацию RTK;
- бортовой вычислитель на Raspberry Pi;
- программную систему автопилота и управления.

На текущем этапе проект находится в стадии подбора компонентов, проектирования архитектуры и подготовки к прототипированию.

---

## Цели проекта

- создать автономную платформу для покоса травы;
- использовать **RTK/GNSS** для точного позиционирования;
- минимизировать зависимость от ручного управления;
- построить систему, которую можно постепенно развивать:
  - от ручного режима;
  - к полуавтоматическому;
  - затем к полностью автономному.

---

## Ключевые задачи

- подбор механической базы для движения по неровной поверхности;
- интеграция косильного модуля;
- реализация надежного питания и силовой части;
- подключение RTK/GNSS-модуля;
- организация связи между базой и управляющим компьютером;
- настройка программной платформы для навигации и маршрутизации;
- обеспечение безопасной остановки и ручного перехвата управления.

## Предполагаемая архитектура

Система строится из нескольких логических частей:

### 1. Мобильная база
Платформа с приводными колесами, способная перемещаться по траве, грунту и неровному рельефу.

### 2. Косильный модуль
Навесной или интегрированный модуль для скашивания растительности.

### 3. Навигация
RTK/GNSS-модуль используется для получения координат с сантиметровой точностью, достаточной для повторяемого движения по участку.

### 4. Бортовой компьютер
Raspberry Pi выступает как центральный управляющий узел:
- принимает данные навигации;
- обрабатывает команды;
- запускает управляющее ПО;
- связывает между собой периферию и исполнительные устройства.

### 5. Канал связи
Радиомодуль / UART bridge нужен для передачи телеметрии, служебных данных и связи между узлами системы.

### 6. Ручное управление
Пульт дистанционного управления нужен для:
- первичной обкатки;
- тестов;
- аварийного перехвата;
- безопасной настройки платформы на ранних этапах.

## Используемые компоненты

## Косильный модуль

Рассматриваются следующие варианты:

- **Flail Mower XFM-1200**
- **ATV120**

Это основные кандидаты на роль рабочего косильного узла для платформы.

## Платформа для тестирования

Для первых экспериментов и отладки используется:

- **Wave Rover**

Эта платформа подходит для ранней проверки:
- связи между узлами;
- логики управления;
- базового движения;
- интеграции вычислительного блока и навигации.

## RTK / GNSS

Для точного позиционирования планируется использовать:

- **UM982 RTK/GNSS**

Этот модуль отвечает за определение координат и работу с RTK-коррекциями.

## Радиомост

Для последовательной связи между устройствами рассматривается:

- **UART radio bridge (Ebyte E220 и аналогичные решения)**

Применение:
- обмен данными между узлами;
- телеметрия;
- резервный канал связи.

## Колеса / привод

Для ходовой части рассматриваются:

- **L-faster Low Speed Agricultural Motor Wheel**

Ожидаемая роль:
- обеспечить высокий крутящий момент;
- устойчивое движение на малой скорости;
- повысить проходимость в условиях травы и рыхлого покрытия.

## Пульт управления

Для ручного режима и тестирования выбран:

- **Microzone MC6C**

Использование пульта особенно важно на ранних этапах сборки и настройки.

## Бортовой компьютер

Основная вычислительная платформа:

- **Raspberry Pi**

На Raspberry Pi планируется разместить:
- управляющее ПО;
- навигационную логику;
- интерфейсы обмена с датчиками и исполнительными узлами.

---

## Программная часть

В качестве программной основы рассматривается:

- **OpenMower**

OpenMower интересен как готовая open-source-платформа для роботизированных косилок с поддержкой RTK-навигации, маршрутизации и автономной работы.

В контексте RoverMower OpenMower может использоваться как:
- отправная точка для архитектуры;
- источник идей по навигации;
- база для адаптации под собственную механику и электронику.


- **ArduPilot Rover**

 
- **ROS2/Nav2**

## Этапы разработки

## Этап 1. Стендовые испытания
- запуск управляющего компьютера;
- подключение RTK/GNSS;
- проверка радиоканала;
- базовое ручное управление.

## Этап 2. Тестовая платформа
- отладка движения на Wave Rover;
- интеграция телеметрии;
- проверка логики команд;
- сбор данных для дальнейшей настройки.

## Этап 3. Полноразмерная база
- перенос электроники на рабочую платформу;
- установка силовой части;
- подключение приводов;
- тестирование устойчивого движения.

## Этап 4. Интеграция косильного модуля
- механическое крепление;
- питание и управление;
- тесты на реальной нагрузке;
- проверка виброустойчивости и надежности.

## Этап 5. Автономное движение
- движение по точкам;
- удержание траектории;
- развороты;
- повторяемые проходы по участку.

## Этап 6. Безопасность и эксплуатация
- аварийная остановка;
- ручной override;
- защита людей и животных;
- безопасный сценарий запуска и остановки.
